# AI 交接文档（AI Handoff）

最近同步：`2026-04-24`  
状态：`active`

## 1. 这份文档的用途

这份文档用于在频繁切换 AI 协作者时，给下一位 AI 一个稳定、统一、低成本的接手入口。

目标不是替代模块文档，而是回答下面 5 个问题：

1. 这个项目整体是什么。
2. 当前已经做到哪一步。
3. 现在最重要的事情是什么。
4. 哪些约束不能被新一轮改动破坏。
5. 下一位 AI 接手后应该先读什么、先看哪里。

## 2. 项目一句话

`ZCG-Raccourci Control` 是一个基于 AutoHotkey v2 的 Windows 全局效率工具，核心同时包含：

- 输入提效器：通过全局热键、悬浮面板、快捷字段实现快速输入
- 扩展容器：通过统一模式切换承载笔记、截图发手机、截图问答、简历自动填写等能力

## 3. 当前稳定入口

- 运行入口：`main.ahk`
- 全局悬浮面板：`Alt+Q`
- 主配置界面：`Alt+Shift+Q`
- 截图问答悬浮窗：`Alt+Shift+A`
- 截图并问答：`F1`

## 6. 当前稳定共识

- 产品定位必须保持“双核心”：输入提效器 + 扩展容器。
- 模式切换入口必须保留，不要把扩展能力做成零散孤立功能。
- Web 配置与本地 AHK 状态必须保持契约一致；涉及保存、读取、模式、热键、Assistant 设置时，要同时检查前后端。
- 每次重要改动后必须同步更新文档，不能只改代码不改文档。

## 7. 当前特别需要新 AI 注意的点

- AHK 开发态会自动热重载；保存 `src/*.ahk` 后，脚本会自动 `Reload()`，调试时要显式考虑这一点。
- 涉及编码、配置写盘、热键映射、Assistant 安全链路的改动都属于高风险区域。
- Assistant 模块最近变化很多，当前稳定口径是：
  - Web 页采用“基础设置 / 高级设置”分层
  - 高级设置默认隐藏，关闭时整块不显示
  - 顶部先展示快捷键说明
  - 模板按钮放回基础设置
  - `API Endpoint` 已从主 Web UI 移除
  - 截图保存目录可直接在主界面修改
  - “最新截图文件”已从主界面移除
  - 透明度改为四档：`20 / 50 / 75 / 100`
- Assistant 悬浮窗当前稳定口径是：
  - 录屏期间本地必须可见
  - 截图时才允许临时隐藏
  - 回答区不再显示“回答区”字样
  - 透明度只允许在主界面修改，悬浮窗内不提供透明度入口
  - 不进入任务栏与常规 Alt-Tab 列表
  - 回答文本从左上角开始排版，并继续做自动换行与代码块整理
- Resume 模块当前稳定口径是：
  - 前端只维护字段名和值
  - 分区说明小字已从主 UI 移除
  - `aliases` 与 `type` 不再放在主 UI 上编辑
  - 后端默认字段表继续维护别名和类型

## 8. 当前优先事项

1. 保持文档与实现继续同步，避免多轮改动后口径漂移。
2. 截图问答模块后续若继续调 UI，优先保持“录屏本地可见 / 截图短隐藏 / WDA 隔离”的安全基线不被回退。
3. 简历自动填写模块后续若继续扩展，优先走后端字段表与插件规则，不要重新把别名和类型复杂度塞回主 UI。

## 9. 已知风险

- 文档量较大，如果不维护交接入口，后续 AI 很容易只看到局部。
- 一些功能同时横跨 AHK、PowerShell、Web 前端，单点修改容易造成契约偏差。
- AHK 热重载会让“代码改了马上生效”和“脚本重载导致状态重建”混在一起，需要注意排障判断。

## 10. 新 AI 接手时建议阅读顺序

1. `README.md`
2. `docs/AI_HANDOFF.md`
3. `docs/COMPONENTS.md`
4. 当前要改动对应的 `docs/modules/*.md`
5. 如涉及共享能力，再看 `docs/shared/*.md`
6. 如涉及接口 / 架构，再看 `docs/components/*.md`
