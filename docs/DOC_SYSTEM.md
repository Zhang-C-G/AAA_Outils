# 文档系统总图（Domain / Module / Shared）

最近同步：`2026-04-22`

## 1. 三层结构

1. `Domain（大块）`：描述业务域目标和边界。
2. `Module（小块）`：可独立维护的功能单元。
3. `Shared（公用块）`：跨多个模块复用的能力与约束。

## 2. 当前 Domain 划分

1. 输入提效域：悬浮面板、字段、提示词、快捷字段。
2. 扩展工作域：笔记、截图发手机、截图问答。
3. 配置平台域：Web 配置前后端、模式切换、快捷键管理。
4. 基础设施域：存储、运行态、日志、主题与样式 Token。

## 2.1 业务模块口径（当前约定）

1. 业务模块固定为 5 个：`快捷字段`、`笔记`、`截图发手机`、`截图问答`、`简历自动填充`。
2. `全局快捷键` 不计入业务模块，归类为 Shared 公用能力。
3. `快捷键` 页面属于“配置平台入口”，用于管理公用热键，不单独计入业务模块数量。

## 3. Shared 公用块清单

1. 全局热键与动态注册。
2. 全局状态与模式切换。
3. 存储契约与配置落盘规则。
4. 日志动作命名与记录规则。
5. UI 样式 Token 与文案一致性规则。

## 4. 目录职责

1. `docs/modules/`：模块级文档（面向具体功能）。
2. `docs/shared/`：公用能力文档（面向复用契约）。
3. `docs/components/`：架构与接口文档（面向全局实现）。
4. `docs/templates/`：文档模板与写作规范。

## 5. 更新顺序（强制）

1. 先更新受影响模块：`docs/modules/*.md`。
2. 涉及公用能力时同步：`docs/shared/*.md`。
3. 涉及架构/API 时同步：`docs/components/*.md`。
4. 记录动作变化：`docs/ACTION_LOG.md`。
5. 记录文档变更：`docs/DOC_CHANGELOG.md`。
