# 文档体系总图
最近同步：`2026-04-25`

## 1. 目标

这份文档用于说明我们当前的主文档体系。
它解决的问题不是某一个功能怎么写，而是：

- 开发时先读什么
- 文档之间是什么关系
- 第一阶段和第二阶段分别对应什么文档
- 全局私人偏好文档放在哪一层
- 检查文档放在哪一层

## 2. 主体系树状图

```text
文档体系
├─ 0. 总入口层
│  └─ AI 开发总指南
│     作用：
│     - 规定开发总流程
│     - 规定第一阶段 / 第二阶段
│     - 规定第二阶段触发时机
│     - 规定通用验收方向
│
├─ 1. 模块层
│  ├─ 模块主文档
│  │  作用：
│  │  - 定义模块目标
│  │  - 定义模块核心功能
│  │  - 记录模块当前状态
│  │  - 作为该模块总入口
│  │
│  └─ 模块下属能力文档
│     作用：
│     - 针对模块中的某个具体能力展开
│     - 例如栏目、悬浮窗、自动保存、快捷键等
│
├─ 2. 第一阶段文档层
│  └─ 第一阶段界面与结构说明
│     作用：
│     - 说明页面结构
│     - 说明入口、布局、区块
│     - 不要求完整业务闭环
│
├─ 3. 第二阶段文档层
│  ├─ 3.1 第二阶段标准能力说明文档
│  │  触发时机：
│  │  - 第一阶段完成后
│  │  - 第二阶段首次真实开发前
│  │
│  │  作用：
│  │  - 先定义这次要实现的能力
│  │  - 按用户点击行为拆颗粒度
│  │  - 定义默认交互、允许行为、禁止行为、持久化要求
│  │
│  ├─ 3.2 第二阶段全局私人偏好文档
│  │  作用：
│  │  - 作为全局私人偏好体系的总入口
│  │  - 先给出分类结构和读取顺序
│  │  - 再按需进入子文档
│  │
│  │  子文档示例：
│  │  - 交互总原则
│  │  - 编辑态与光标偏好
│  │  - 同构结构偏好
│  │  - 自动保存与保存按钮偏好
│  │
│  ├─ 3.3 第二阶段最终能力说明文档
│  │  来源：
│  │  - 标准能力说明文档
│  │  - 全局私人偏好文档
│  │    融合后形成
│  │
│  │  作用：
│  │  - 作为最终实现依据
│  │  - 后续代码开发按这份文档执行
│  │
│  └─ 3.4 第二阶段检查分支
│     作用：
│     - 检查实现是否符合最终能力说明文档
│     - 检查是否有漏项
│     - 检查是否出现假保存、假加载、假交互
│
├─ 4. 检查层
│  ├─ 4.1 通用检查文档
│  │  作用：
│  │  - 检查模块整体闭环
│  │  - 刷新、重启、持久化、前后端同步、日志、回归
│  │
│  └─ 4.2 高频能力专项检查文档
│     作用：
│     - 检查某一类高频能力是否完整
│     - 例如：
│        - 栏目能力检查
│        - 悬浮窗能力检查
│        - 快捷键能力检查
│        - 自动保存能力检查
│        - 拖拽排序能力检查
│
├─ 5. 偏好约束层
│  └─ 全局私人偏好文档
│     作用：
│     - 作为整个项目的长期偏好源头
│     - 给所有模块提供统一实现风格约束
│
└─ 6. 归档与恢复层
   ├─ AI 交接文档
   │  作用：
   │  - 记录当前做到哪里
   │  - 记录已实现 / 未实现 / 风险 / 回滚点
   │
   └─ 问题归档 / 事件归档

      - 记录历史问题
      - 记录原因
      - 记录以后需要避免重复犯错的点
```

```text

```

## 4. 关键理解

### 4.1 第一阶段

第一阶段负责把界面壳层搭出来。
重点是：

- 页面结构
- 模块入口
- 布局区块
- 基础说明

这一阶段不要求完整业务闭环。

### 4.2 第二阶段

第二阶段不是直接写代码。
第二阶段的第 1 步必须先写能力说明文档。

而且第二阶段内部还要再分层：

1. 标准能力说明
2. 全局私人偏好总索引
3. 相关主题子文档
4. 融合后的最终能力说明
5. 实现后检查

### 4.3 私人偏好文档

私人偏好文档不是替代标准能力说明。
它是在标准能力说明之后增加的一层约束。

它主要解决的问题是：

- 功能没有错，但实现风格不符合个人习惯
- 默认交互虽然能用，但不符合长期偏好

例如：

- 栏目改名要原位编辑
- 不允许点击后外框变大
- 不喜欢突变式编辑态

这类内容应进入全局私人偏好体系中的对应主题子文档，
再统一融合进最终能力说明。
