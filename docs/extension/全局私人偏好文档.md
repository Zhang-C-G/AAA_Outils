# 全局私人偏好文档

最近同步：`2026-04-26`

## 1. 文档定位

这份文档现在不再承担“把所有私人偏好细则都堆在一个文件里”的职责。

它只负责四件事：

1. 说明这套全局私人偏好体系是干什么的
2. 说明进入第二阶段时应该怎么读取
3. 给出当前的分层结构和入口
4. 约束后续新增偏好时应该写到哪里

也就是说：

- 本文档负责“总索引”和“总入口”
- 具体偏好细则拆到子文档中维护
- 最终模块方案应把“标准能力说明 + 全局私人偏好结论”融合成模块自己的最终能力说明

## 2. 读取顺序

每次进入第二阶段时，建议按这个顺序读取：

1. `docs/extension/AI_DEVELOPMENT_PLAYBOOK.md`
2. 对应模块文档 `docs/modules/*.md`
3. 该模块本次标准能力说明文档
4. 本文档 `docs/extension/全局私人偏好文档.md`
5. 先读“通用偏好结论层”
6. 再按需要读“原则层”和“控件层”
7. 最终融合为该模块的最终能力说明文档

## 4. 分层说明

### 4.1 总索引层

- 文件：`docs/extension/全局私人偏好文档.md`
- 作用：
  - 告诉后续开发这套文档怎么读
  - 说明当前有哪些层
  - 给出入口，不展开全部细则

### 4.2 通用偏好结论层

- 文件：`docs/extension/global_preferences/00_通用偏好结论.md`
- 作用：
  - 不是原始细则
  - 而是把多个主题偏好综合后，提炼成“默认执行结论”
- 适用内容：
  - 默认交互应该偏向什么
  - 默认编辑态应该如何处理
  - 默认保存策略应该如何选择
  - 默认结构类控件应该如何取舍

### 4.3 原则层

- 目录：`docs/extension/global_preferences/`
- 当前文件：
  - `01_交互总原则.md`
  - `02_编辑态与光标偏好.md`
  - `03_同构结构偏好.md`
  - `04_自动保存与保存按钮偏好.md`
- 作用：
  - 存放抽象规则
  - 解释为什么形成某种通用偏好结论

### 4.4 控件专项层

- 目录：`docs/extension/global_preferences/components/`
- 当前文件：
  - `10_栏目框偏好.md`
  - `11_按钮偏好.md`
  - `12_标签页偏好.md`
- 作用：
  - 面向具体 UI 对象
  - 把抽象偏好落实到控件设计与交互细节上

## 6. 维护规则

后续新增全局私人偏好时，不要直接无序堆到本文档里。

推荐规则：

1. 先判断它属于哪一层：
   - 总索引层
   - 通用结论层
   - 原则层
   - 控件专项层
2. 再判断它属于哪个主题或哪个控件
3. 写入对应子文档
4. 如果现有主题不适合，再新增新的子文档
5. 最后只在本文档补索引，不在本文档重复展开长篇细则

## 7. 当前状态

目前已明确：

1. 全局私人偏好属于全局层，而不是单模块私有层
2. 当前体系采用“总索引 + 通用结论 + 原则层 + 控件层”的结构
3. 第二阶段默认先读总索引，再读通用结论，再按需深入到对应原则和控件文档
