# 全局私人偏好：交互总原则
最近同步：`2026-04-25`

## 1. 定位

本文件记录全项目通用的交互总原则。
这类规则优先级高于具体样式偏好，但仍低于模块核心主功能。

## 2. 具体规则

### 2.1 核心优先

- 一切偏好都不能凌驾于模块核心主功能之上
- 如果偏好和核心能力冲突，先保核心，再谈偏好

### 2.2 低扰动优先

- 高频操作区域优先采用稳定、直接、克制的交互
- 不追求花哨变化，不制造无意义视觉跳动

### 2.3 直觉入口优先

- 用户最自然会点的地方，就应该优先作为默认入口
- 不把高频操作藏进二级菜单、右键菜单或额外按钮

例如：

- 常用切换优先直接点击当前结构本体
- 需要高频触达的动作，不优先塞进额外层级
