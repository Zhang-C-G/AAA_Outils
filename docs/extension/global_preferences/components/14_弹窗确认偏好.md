# 全局私人偏好：弹窗确认偏好

最近同步：`2026-04-26`

## 1. 文档定位

适用于删除、清空、覆盖、重置这类需要用户再次确认的操作。

## 2. 默认偏好

### 2.1 确认方式

- 需要再次确认时，优先使用应用内部自定义弹窗
- 不使用浏览器默认 `confirm()`
- 弹窗风格要与当前界面保持一致，不能突然跳出系统原生样式

### 2.2 删除链路

- 删除操作只做一次确认
- 这一次确认要直接、清楚、可执行
- 确认文案应明确指出删除对象

### 2.3 视觉与交互

- 弹窗要聚焦当前动作，不要做多余说明
- 危险操作的确认按钮可以使用危险态，但整体界面不要过度高亮
- 支持键盘确认与取消，保证操作连续性

## 3. 禁止倾向

- 不喜欢默认浏览器确认框破坏整体 UI
- 不喜欢删除时连续弹两次确认
- 不喜欢文案模糊，看不出到底删的是哪个对象
