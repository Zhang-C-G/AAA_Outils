# 全局私人偏好：栏目框偏好

最近同步：`2026-04-26`

## 1. 文档定位

适用于栏目框、栏目标签、可切换栏目容器这类结构。

## 2. 默认偏好

### 2.1 视觉

- 栏目框整体应稳定、紧凑、易扫视
- 激活态要明确，但不靠夸张放大或跳动制造差异
- 未激活态应足够克制，不与主内容争抢注意力

### 2.2 交互

- 单击默认用于切换栏目
- 双击默认用于改名
- 删除应走确认链路
- 拖拽排序应尽量直接，不额外引入复杂模式切换

### 2.3 编辑态

- 改名优先原位编辑
- 编辑时尽量保持栏目框尺寸稳定
- 不默认全选原文字
- 光标默认放在文本末尾
- 双击进入改名态时，栏目框宽高保持不变
- 改名态默认只改变外圈边框颜色，边框切为白色即可
- 不通过整体放大、额外外扩、明显描边增厚来提示“正在编辑”

### 2.4 结构规则

- 同组栏目默认按同一套逻辑处理
- 不轻易给某个栏目设置特殊交互
- 如果确实需要内置栏目特权，必须在模块文档中明确说明
- 栏目切换属于个性化状态，默认应记住用户上次选中的栏目
- 重新打开时，优先恢复上次选中的栏目，而不是强制退回默认第一栏

### 2.5 编辑态贴合补充

- 栏目进入改名态时，文字输入框应尽量完全贴合外部栏目框
- 不希望出现“外框里又缩进一层小输入框”的观感
- 输入框可以保留最基础的文字留白，但视觉上必须与外框对齐、贴边、同圆角
- 如果需要取舍，优先保证“编辑框像原栏目框本体的一部分”，而不是“像额外叠上去的第二层控件”

## 3. 禁止倾向

- 不喜欢点击后尺寸突然变大
- 不喜欢进入编辑态后像换了一个控件
- 不喜欢同组栏目里只有某一项拥有隐藏特权，但文档里没说明
- 不喜欢进入改名态后整块栏目框放大或挤压邻近栏目
- 不喜欢改名态靠多层高亮，只希望外圈颜色切换即可
- 不喜欢改名态时输入框与外部栏目框之间还有明显空隙
