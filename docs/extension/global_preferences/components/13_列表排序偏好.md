# 全局私人偏好：列表排序偏好

最近同步：`2026-04-26`

## 1. 文档定位

适用于笔记列表、快捷字段列表、条目罗列框这类“同层级项目可重排”的结构。

## 2. 默认偏好

### 2.1 交互

- 列表项允许直接拖拽改顺序
- 不希望为了排序再进入额外的“管理模式”或“排序模式”
- 拖拽反馈应轻量明确，能看出当前正在拖拽和目标位置即可

### 2.2 持久化

- 拖拽后的顺序默认应持久化
- 重新进入模块时，应优先恢复用户上次手动排好的顺序
- 顺序属于个人偏好的一部分，不应被无关刷新或临时加载覆盖

### 2.3 适用对象

- 笔记模块中的左侧罗列框
- A 模块中的快捷字段条目顺序
- 其他未来出现的个人条目清单，只要本质上属于“用户自己管理顺序”的列表，也默认继承本偏好

## 3. 禁止倾向

- 不喜欢只能通过删除重建来变相调整顺序
- 不喜欢拖拽后只在当前页生效，重开就丢
- 不喜欢为了排序引入多余按钮、额外弹窗或复杂说明
