# 全局私人偏好：同构结构偏好
最近同步：`2026-04-25`

## 1. 定位

本文件适用于栏目、分区、标签、列表项这类“本质相同、重复出现”的结构。

## 2. 具体规则

### 2.1 同构结构尽量统一规则

- 如果一组结构本质相同，就尽量执行同一套交互逻辑
- 不轻易人为制造“默认项 / 内置项 / 特权项”三套规则

说明：

- 例如栏目、分区、标签这类同构结构，默认应优先统一增删改拖逻辑
- 若某模块确实需要特权项，必须在该模块能力说明里明确写出原因，而不是默认偷偷存在

### 2.2 单击切换优先于单击改名

- 对承担“切换”职责的结构，单击优先保留给切换
- 改名这类编辑动作，如果容易和单击切换冲突，优先采用双击进入

说明：

- 这条不是所有结构的硬编码规则，而是同构切换结构的默认处理方向
- 若某模块有更强约束，应在该模块能力说明中明确说明
