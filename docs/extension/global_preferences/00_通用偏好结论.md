# 全局私人偏好：通用偏好结论

最近同步：`2026-04-26`

## 1. 文档定位

这份文档不是原始细则集合，而是把当前各类全局私人偏好综合后，提炼成默认执行结论。

作用是：

- 给后续实现一个“先看这页就能知道大方向”的入口
- 减少每次都要在多个偏好文档之间来回跳
- 让模块最终能力说明有统一的默认参照

## 2. 默认结论

### 2.1 默认交互结论

- 优先低打扰，不追求无意义的闪动和突变
- 优先让用户在最直觉的位置完成操作
- 高频操作尽量留在主路径，不藏进深层入口

### 2.2 默认编辑结论

- 优先原位编辑
- 优先保持尺寸稳定
- 不默认全选原文字
- 光标默认放在文本末尾，便于继续修改
- 编辑态输入框应尽量贴合原外框，不要再出现一层明显内缩的小框

### 2.3 默认结构结论

- 同构结构优先采用同一套规则
- 不轻易制造“默认项 / 内置项 / 特权项”三套逻辑
- 承担切换职责的结构，默认单击切换
- 改名等编辑动作，如与单击切换冲突，默认双击进入
- 栏目框这类切换结构在进入改名态时，默认保持原框大小不变
- 改名态默认只允许外圈边框颜色切为白色，不靠放大或形态突变提示状态
- 对于栏目框这类原位编辑控件，默认要求输入框与外框贴合、同圆角、同边界
- 对于笔记列表、快捷字段列表这类同层级项目，默认允许直接拖拽调整顺序

### 2.4 默认保存结论

- 持续编辑型模块优先自动保存
- 保存按钮不是默认必须存在
- 是否保留保存按钮，取决于真实保存链路是否有可靠兜底
- 对于栏目、子栏目这类个性化选择，默认也应持久化保存
- 重新进入模块时，优先恢复用户上次停留的位置
- 对于用户主动拖拽出来的列表顺序，默认也应持久化保存并在重进时恢复

### 2.5 默认控件结论

- 栏目框、标签页、按钮这类高频控件，优先稳定、克制、易扫视
- 激活态要明确，但不应靠夸张跳变来制造存在感
- 危险操作要有明确识别，但不要让整页长期处于“危险高亮”噪音中
- 需要再次确认的操作，默认使用应用内自定义弹窗，不使用浏览器默认 `confirm()`
- 删除动作默认只做一次确认，不叠加第二次确认
- 快捷键说明条、同步说明条、只读提示条这类辅助说明区域，默认允许使用低对比斜杠纹理背景来增强识别

## 3. 使用方式

如果一个模块暂时没有更具体的私人偏好约束，就默认按本文档的结论执行。

如果模块需要例外，必须：

1. 在模块能力说明中明确写出例外原因
2. 说明为什么默认结论不适用
3. 避免只在实现里偷偷偏离
