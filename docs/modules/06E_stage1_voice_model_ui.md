# E模块第一阶段：问答/语音模型拆分UI

## 目标

- 将 E 模块里的模型配置拆成“问答模型”和“语音模型”两组 UI。
- 保持现有页面结构不推翻，只补第一阶段需要的字段、占位状态和前端回填。
- 不在本阶段接入后端存储、运行时调用和真实 API 链路。

## 本阶段范围

- 基础设置：
  - `模型选择` 改为 `问答模型选择`
  - 新增 `语音模型选择`
- 高级设置：
  - `API Key` 改为 `问答模型 API`
  - 新增 `语音模型激活`
  - 新增 `语音模型 API`
- 前端状态：
  - 增加语音模型相关默认值
  - 增加页面刷新后的回填
  - 增加自动保存读取这些字段的前端占位逻辑

## 本阶段明确不做

- 不新增 AHK 配置读写
- 不新增后端 settings 持久化
- 不接真实语音转文字 API
- 不改麦克风采集链路
- 不改截图问答主链路

## 页面结构

- 基础设置区保留高频项：
  - 问答模型选择
  - 语音模型选择
  - 模板选择
  - 模板名称
  - Prompt
- 高级设置区保留低频项：
  - 透明度
  - 增强模式
  - 问答模型 API
  - 禁止复制
  - 限流
  - 语音识别功能
  - 语音模型激活
  - 语音模型 API
  - 麦克风选择

## 第二阶段待补

1. 语音模型选择写入后端 settings 与 `config.ini`
2. 语音模型 API 的保存、掩码回填与运行时读取
3. 语音模型激活对真实语音链路的生效逻辑
4. 语音模型候选列表由后端统一下发，而不是只用前端占位
