# 模块 04：B 笔记

## 模块目标

- 管理多条笔记（新建、编辑、删除）。
- 输入过程中自动保存当前编辑内容。
- 支持 Markdown 内容写入，并稳定落盘到本地笔记文件。

## 核心主功能

- 核心是笔记内容可编辑、可保存、可自动保存，且刷新后不丢。
- 任何界面调整或次级交互问题，都不能以牺牲自动保存链路为代价。
- “笔记显示”是独立模块，不属于本模块；两者现在也不共享同一套笔记文件。

## 当前状态

- 当前状态：`未开发完全`
- 目前已经完成的是基础的笔记新建、编辑、删除、自动保存。
- 仍不能把本模块视为最终完成版；后续若继续扩展，必须在不破坏现有保存链路的前提下推进。

## 当前默认交互

- 不再提供“保存笔记”独立按钮。
- 标题或正文输入后，前端延迟自动保存。
- 切换笔记、切换模块、刷新前草稿恢复，仍然作为自动保存链路的兜底。
- 这属于当前项目的私人习惯，后续如继续扩展 B 模块，默认仍按“自动保存优先、手动保存按钮缺省”处理。

## 改动后必查

1. 新建笔记后可保存并刷新可见。
2. 输入标题或正文后，不点任何保存按钮也能自动落盘。
3. 删除确认流程正常且落盘生效。
4. Markdown 内容可正常保存到 `notes/*.md`。
5. 刷新页面或重启后，笔记编辑内容不丢失。
6. 切换到其他笔记前，当前未落盘内容会先自动保存。
7. 当前笔记标题修改后，左侧列表标题能同步刷新。

## 第二阶段检查结论

### 已对齐项

1. 前端输入、后端保存、本地 `notes/*.md` 落盘链路已接通。
2. B 模块已改为自动保存优先，不再依赖独立“保存笔记”按钮。
3. 切换笔记、切换模块、原生刷新草稿恢复，这三条链路仍在。
4. B 模块与 C 模块仍保持独立，不共用同一套笔记文件。

### 本轮补充修正

1. 删除了 B 模块页面内的“保存笔记”按钮，避免和默认自动保存逻辑冲突。
2. 补齐了标题与正文输入后的延迟自动保存，不再只靠切模式时保存。
3. 标题编辑时，左侧笔记列表会同步更新，减少“改了标题但列表没变”的错觉。
