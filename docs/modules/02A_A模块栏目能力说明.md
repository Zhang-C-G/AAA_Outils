# A 模块栏目能力说明
最近同步：`2026-04-25`

## 1. 文档定位

这份文档属于 A 模块在第二阶段中的“能力说明文档”。
它不负责讲整个模块的所有事情，而是只负责定义：

- A 模块里的“栏目”应该具备什么能力
- 用户对栏目做每一步操作时，系统应该怎么响应
- 哪些行为允许，哪些行为禁止
- 哪些结果必须持久化

后续实现、排障、验收，都应该以这份文档为准。

## 2. 该能力的目标

A 模块中的栏目，负责承载不同字段集合的分类入口。
栏目能力的目标不是“显示几个标签”，而是要让用户可以稳定地：

- 新增栏目
- 切换栏目
- 修改栏目标题
- 删除栏目
- 拖动栏目排序
- 在刷新和重启后仍保持一致

## 3. 用户操作颗粒度说明

### 3.1 用户点击“新增栏目”

系统应当：

- 立即新增一个栏目
- 新栏目默认标题为 `新栏目`
- 自动切换到该栏目
- 该栏目对应的数据区立即可编辑
- 该变化进入自动保存链路

边界要求：

- 新增栏目后不能只显示标签，不创建对应数据容器
- 新增栏目后刷新不能直接消失

### 3.2 用户点击某个栏目

系统应当：

- 切换当前选中栏目
- 右侧数据区切换到该栏目对应内容
- 不应误触发改名
- 不应破坏拖拽状态

边界要求：

- 切换栏目时，如果当前栏目存在未保存编辑，应继续走自动保存链路
- 切换后不能出现标题切了但内容还停留在旧栏目

### 3.3 用户双击栏目标题

默认交互要求：

- 单击栏目时，只执行栏目切换
- 双击栏目标题，才进入改名状态
- 输入框应自动获得焦点
- 输入框不默认全选
- 光标默认放在当前文本末尾，便于在原标题基础上继续修改

边界要求：

- 双击标题进入改名时，不应误触发拖拽
- 双击标题进入改名时，如果栏目尚未选中，应先选中再进入改名

### 3.4 用户在栏目改名状态下按 `Enter`

系统应当：

- 提交新标题
- 退出改名状态
- 保持当前栏目仍为选中状态
- 进入自动保存链路

边界要求：

- 空标题不能提交为有效名称
- 标题未变化时，不应制造无意义保存

### 3.5 用户在栏目改名状态下失焦

系统应当：

- 默认按提交处理
- 如果输入内容有效，则保存新标题
- 如果输入为空，则回退到原标题

### 3.6 用户在栏目改名状态下按 `Escape`

系统应当：

- 取消本次改名
- 恢复原标题
- 退出改名状态

### 3.7 用户点击“删除当前栏目”

系统应当：

- 允许删除当前栏目
- 删除前给出确认
- 删除后自动切换到仍然存在的栏目
- 删除后进入自动保存链路

禁止行为：

- 只剩最后一个栏目时，不允许删除到 0 个栏目

### 3.8 用户拖动栏目

系统应当：

- 允许拖动栏目改变顺序
- 松手后立即更新顺序显示
- 顺序进入自动保存链路

边界要求：

- 拖动不应改变栏目内容归属
- 拖动后刷新应保持同样顺序
- 重启后顺序应保持一致

## 4. 持久化要求

栏目能力至少要持久化以下结果：

1. 栏目是否存在
2. 栏目标题
3. 栏目顺序
4. 栏目对应的数据内容

如果这些只在前端内存里变化，而没有写入本地配置，就不算完成。

## 5. 允许与禁止规则

### 5.1 允许

- 允许新增栏目
- 允许双击栏目标题直接改名
- 允许删除栏目
- 允许拖动栏目排序

### 5.2 禁止

- 禁止把空标题保存成有效栏目名
- 禁止保存重复栏目名
- 禁止拖动后只改显示、不改真实顺序
- 禁止改标题后丢失栏目下已有数据
- 禁止栏目保存被不相关的快捷键校验误伤

## 6. 常见风险点

### 6.1 假保存

表现：

- 前端看起来已经改名
- 刷新后恢复原状

说明：

- 标题只改了前端状态，没有真正写入后端和本地配置

### 6.2 假排序

表现：

- 拖动后眼前顺序对了
- 刷新后顺序又回去

说明：

- 顺序没有持久化

### 6.3 假新增

表现：

- 新栏目出现了
- 再次加载后消失

说明：

- 栏目容器或栏目本身没有写入本地配置

### 6.4 假改名

表现：

- 改名时看起来成功
- 实际只是临时 DOM 输入框替换
- 刷新后回到旧名字

说明：

- 改名没有进入真实保存链路

### 6.5 假解耦

表现：

- 改栏目名、删栏目、拖栏目时
- 实际报错却来自快捷键配置

说明：

- 栏目能力没有和整页其他配置校验解耦
- 导致不相关配置阻塞栏目能力保存

## 7. 本能力的最低验收项

1. 新增栏目后，刷新仍存在
2. 双击栏目标题可直接改名
3. 改名后刷新仍保留新标题
4. `Escape` 取消改名有效
5. 栏目可删除
6. 任意栏目标题都可通过双击改名
7. 任意栏目都可删除，但不能删到一个不剩
8. 不允许保存重复栏目名
9. 栏目可拖动排序
10. 拖动后刷新顺序不变
11. 改标题、删除、拖动都不会破坏栏目下已有内容
12. 栏目操作不会被不相关快捷键校验阻塞
