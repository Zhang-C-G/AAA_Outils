# A 模块栏目最终能力说明
最近同步：`2026-04-25`

1. 标准能力说明文档
2. 全局私人偏好文档

后续实现、修正、验收，应优先以这份文档为准。

## 3. 最终目标

A 模块中的栏目，不仅要功能完整，还要交互稳定、低扰动。

最终目标包括两层：

### 3.1 标准能力目标

- 新增栏目
- 切换栏目
- 修改栏目标题
- 删除栏目
- 拖动栏目排序
- 在刷新和重启后保持一致

### 3.2 私人偏好目标

- 改名必须原位编辑
- 进入编辑态时不允许尺寸突变
- 交互变化必须轻，不突兀
- 栏目交互整体保持低扰动
- 栏目不设默认特权，所有栏目执行同一套逻辑

## 4. 最终交互规则

### 4.1 新增栏目

系统应当：

- 立即新增栏目
- 默认标题为 `新栏目`
- 自动切换到该栏目
- 立即创建对应数据容器
- 进入自动保存链路

### 4.2 点击栏目

系统应当：

- 切换当前栏目
- 更新右侧数据区
- 不误触发改名
- 不破坏拖拽状态

### 4.3 双击栏目标题

系统应当：

- 单击栏目时，只执行栏目切换
- 双击栏目标题时，才进入改名状态
- 在原有栏目框架内编辑
- 尺寸尽量保持不变
- 输入内容不默认全选
- 光标默认放在当前文本末尾
- 编辑态视觉变化尽量轻

禁止行为：

- 不允许点击后栏目外框明显变大
- 不允许点击后栏目高度明显突变
- 不允许替换成风格完全不同的编辑控件

### 4.4 在改名状态下按 `Enter`

系统应当：

- 提交新标题
- 退出改名状态
- 保持当前栏目为选中状态
- 进入自动保存链路

### 4.5 在改名状态下失焦

系统应当：

- 默认按提交处理
- 输入有效则保存
- 输入为空则回退到原标题

### 4.6 在改名状态下按 `Escape`

系统应当：

- 取消改名
- 恢复原标题
- 退出改名状态

### 4.7 删除栏目

系统应当：

- 允许删除当前栏目
- 删除前确认
- 删除后自动切换到有效栏目
- 删除后进入自动保存链路
- 不允许删到一个栏目都不剩

### 4.8 拖动栏目

系统应当：

- 允许拖动排序
- 松手后立即更新顺序
- 顺序进入自动保存链路

边界要求：

- 不改变栏目下数据归属
- 刷新后顺序一致
- 重启后顺序一致

## 5. 最终禁止规则

1. 禁止保存空栏目名
2. 禁止保存重复栏目名
3. 禁止拖动后只改显示不改真实顺序
4. 禁止改名后丢失栏目下已有数据
5. 禁止栏目操作被不相关快捷键校验阻塞
6. 禁止栏目改名时出现尺寸突变
7. 禁止栏目改名时出现明显的组件替换感
8. 禁止把栏目人为分成“默认栏目 / 内置栏目 / 普通栏目”三套逻辑
9. 禁止删除时把栏目删到一个不剩

## 6. 最终验收项

1. 新增栏目后刷新仍存在
2. 任意栏目标题双击后都可直接改名
3. 任意栏目都执行同一套规则，不存在默认特权栏目
4. 改名过程保持原位编辑
5. 改名时栏目尺寸无明显突变
6. `Enter / Escape / 失焦` 行为符合预期
7. 不允许保存重复栏目名
8. 任意栏目可删除，但不能删到一个不剩
9. 栏目可拖动排序，刷新后顺序不变
10. 栏目操作不会被不相关快捷键校验阻塞
